# Agent Spec — MCP-Tripwire ADK layer

> Codified per the `agents-cli` workflow Phase 0 convention.
> See [AGENTS.md](AGENTS.md) for project-wide rules; this spec covers the
> non-deterministic ADK layer specifically.

## Overview

MCP-Tripwire is a trust gateway for MCP tools. The deterministic core
(scanner, engine, attestation, OWASP map, proxy bridge) does the actual
enforcement — it is stdlib-only by Hard Rule #2 and runs without any model.

The ADK layer adds three cooperating agents on top of that core, so a human
operator can hold a natural-language conversation about a target MCP server
and get back an explainable verdict:

- **Scanner** classifies a target server's tool descriptors and explains
  the findings in plain language, mapped to the OWASP MCP Top 10.
- **Red-team** proposes adversarial probes (seeded from the shipped corpus)
  so the operator can stress-test their gateway.
- **Attestor** approves and issues a signed trust badge, gated by
  human-in-the-loop confirmation for high-stakes attestations.

Each agent wraps a deterministic core function as an ADK `FunctionTool`,
so every tool call has a verifiable, reproducible outcome — the LLM only
provides routing and explanation, never the decision.

## Example Use Cases

| Operator says | Coordinator routes to | Tool called | Result |
|---|---|---|---|
| "Scan this tool: `{...}`" | Scanner | `scan_tool_descriptor` | Findings grouped by OWASP category, with the rule that fired and a one-line evidence snippet. |
| "Approve and badge `get_weather`" | Attestor | `issue_if_clean` (confirmation required) | Either a signed trust badge or an explanation of which finding blocked it. |
| "Show me probes I should be catching" | Red-team | `seed_probes` | The 8 attack cases from `corpus/attacks.jsonl` plus the rug-pull case, with their OWASP category. |
| "Re-evaluate the badge for `weather` — the description changed" | Scanner → Attestor | `scan_tool_descriptor` then `issue_if_clean` | If the new descriptor still passes, a fresh badge; otherwise the quarantine reason. |

## Tools Required

| Tool | Module | Auth | Returns |
|---|---|---|---|
| `scan_tool_descriptor(tool: dict)` | `tripwire.agents.scanner_agent` | none | `{"findings": [...], "owasp_categories": [...], "worst_severity": "..."}` |
| `seed_probes()` | `tripwire.agents.redteam_agent` | none | `{"probes": [...], "count": N, "source": "corpus/attacks.jsonl"}` |
| `propose_probe(category: str)` | `tripwire.agents.redteam_agent` | none | One probe from the corpus matching the category, or a default. |
| `issue_if_clean(tool: dict)` | `tripwire.agents.attestor_agent` | `TRIPWIRE_SIGNING_KEY` env | `{"action": "...", "badge": {...} | None, "findings": [...]}` |

All tools call into the deterministic core (`tripwire.detection`,
`tripwire.engine`, `tripwire.corpus`). No external network calls.

## Constraints & Safety Rules

- **Hard Rule #2** — `google.adk` imports only under `src/tripwire/agents/`
  and `app/`. Enforced by `scripts/harness_guardrails.py`.
- **Hard Rule #3** — signing key from `TRIPWIRE_SIGNING_KEY`, never hardcoded.
- **Hard Rule #4** — any demo data uses the canary secret + local fake sink;
  the agents do not touch `~/.ssh`, env, or real credentials.
- **Hard Rule #6** — Scanner never invents findings; its tool returns
  exactly what `tripwire.detection.scan_tool` produced.
- **Attestor requires confirmation** — `FunctionTool(issue_if_clean,
  require_confirmation=True)` so badge issuance is human-gated.
- **The LLM never decides allow/block.** Verdicts are computed by the
  deterministic engine; the LLM explains them.

## Success Criteria

- `agents-cli info` reports the project correctly (already true).
- `python -c "from app.agent import root_agent, app"` imports cleanly when
  the `[agent]` extra is installed (covered by `tests/unit/test_agents.py`).
- `make demo-adk` runs end-to-end without an LLM call, exercising the
  deterministic tool surface of each agent. Exit code 0.
- `make check` stays green; no third-party imports leak into the
  deterministic core (Hard Rule #2 verified by guardrails).
- Eval (`agents-cli eval`) — wired in a follow-up issue; this PR keeps
  the existing `tests/eval/datasets/{tool_poisoning,schema_drift}.json`
  in place and ensures `eval_config.yaml` still references them.
